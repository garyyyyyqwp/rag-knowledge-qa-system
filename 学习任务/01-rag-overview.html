第 01 课

RAG 技术概述：为什么大模型需要外部知识

从 LLM 的局限性出发，理解检索增强生成的核心理念与工作流程

01

什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与大语言模型生成相结合的技术架构。它的核心思想是：在 LLM 回答问题之前，先从外部知识库中检索最相关的文档片段，将其注入到 Prompt 中，使 LLM 能够基于"参考资料"生成更准确、更可信的回答。

R = Retrieval（检索）

从知识库中查找相关信息。将用户查询向量化后，在向量数据库中执行相似度搜索，返回语义最相近的文档片段。

A = Augmentation（增强）

将检索结果注入 Prompt。把搜索到的相关文档片段拼接到提示词模板中，为 LLM 提供可靠的参考上下文。

G = Generation（生成）

LLM 基于增强后的上下文生成回答。模型不再仅凭内部记忆作答，而是引用外部知识，大幅降低幻觉风险。

RAG 核心流程：用户提问 → 向量检索 → 相关文档 → 拼接上下文 → LLM 生成 → 带引用回答

02

为什么 LLM 需要 RAG？

大语言模型虽然能力强大，但存在三个核心局限性，而 RAG 正是为弥补这些不足而生：

幻觉问题（Hallucination）

LLM 可能编造不存在的事实、人物或事件。当被问及超出训练数据的知识时，模型仍会"自信地"给出错误答案。RAG 通过提供可验证的外部文档来抑制幻觉。

知识截止日期（Knowledge Cutoff）

训练数据有截止时间。例如 GPT-4 无法知晓训练之后发生的事件、发布的新论文或最新的法规。RAG 连接到实时文档库，使模型能够回答关于最新信息的问题。

领域知识缺失（Domain Knowledge Gap）

通用模型缺乏特定领域的专业知识，如企业内部文档、医疗病例、法律判例等。RAG 可以将这些私有知识注入上下文，使通用模型也能胜任专业任务。

RAG 的核心价值 让 LLM "引用"外部知识，而不是仅凭记忆回答。正如学生考试时"开卷"比"闭卷"更可靠——RAG 就是给 LLM 的"开卷考试"。
03

RAG 工作流程 — 四阶段详解

一个完整的 RAG 系统包含四个阶段，每个阶段都至关重要：

  1. 索引（Indexing）

    文档 → 文本提取 → 分块（Chunking）→ 向量化（Embedding）→ 存入向量数据库。这一阶段将非结构化的文档转化为可检索的向量表示。

    本项目使用：pdfplumber 进行 PDF 文本提取，OpenAI Embedding API 生成向量，ChromaDB 存储与检索。

  2. 检索（Retrieval）

    用户提问 → 向量化 → 相似度搜索（余弦相似度）→ 返回 Top-K 个最相关的 Chunk。检索质量直接决定了 RAG 的上限。

  3. 增强（Augmentation）

    将检索到的 Chunk 拼入 Prompt 模板中。典型的 Prompt 结构为：系统指令 + 检索到的上下文 + 用户原始问题。

  4. 生成（Generation）

    LLM 基于增强后的 Prompt 生成答案，并在回答中标注信息来源（Citation），让用户能够验证内容的真实性。

四阶段流水线：索引 → 检索 → 增强 → 生成

04

RAG 的实际应用场景

RAG 已经在各行各业得到广泛应用，以下是最典型的几个场景：

场景 说明 典型案例
智能客服 将产品手册、FAQ 文档向量化，客服机器人可精准回答用户问题，避免编造不存在的功能。 电商售后、银行客服、SaaS 平台帮助中心
法律咨询 将法律法规、判例文档接入 RAG 系统，辅助律师快速查找相关法条和过往案例。 法律文书检索、合同审查
医疗问答 将医学文献、药品说明书接入知识库，辅助医生或患者获取有据可查的医学建议。 临床决策支持、用药咨询
个人知识库 将个人的笔记、日记、阅读材料向量化，打造专属的"第二大脑"。 Obsidian + RAG、Notion AI
企业文档搜索 将企业内部 Wiki、技术文档、会议纪要接入 RAG，实现跨部门知识共享。 Confluence + RAG、企业知识中台

本课小结

  • RAG 由检索（Retrieval）、增强（Augmentation）、生成（Generation）三部分组成，本质是给 LLM 加上外部知识库
  • LLM 的三大局限——幻觉、知识截止日期、领域缺失——正是 RAG 要解决的核心问题
  • RAG 的四个阶段：索引 → 检索 → 增强 → 生成，每一阶段都有其独特的技术挑战
  • RAG 已广泛应用于智能客服、法律咨询、医疗问答、企业知识管理等场景
  • 本课程将带你从 Embedding 基础开始，逐步构建完整的 FastAPI + ChromaDB + OpenAI RAG 系统
