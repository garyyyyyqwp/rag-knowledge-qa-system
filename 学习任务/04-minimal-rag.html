最简 RAG 管道：Embed → Retrieve → Generate

不依赖任何向量数据库，用纯 Python 实现完整的 RAG 流程

01

目标：从零构建 RAG

在深入了解 ChromaDB 等专业工具之前，我们先用手边最简单的工具——纯 Python + NumPy + OpenAI API——构建一个完整但极简的 RAG 管道。整个流程包括：

10 条知识片段
→
文本向量化
→
内存向量搜索
→
LLM 生成回答

不需要安装 ChromaDB、Milvus 或任何向量数据库——所有向量都存储在 Python 的 list 中，搜索就靠 NumPy 的余弦相似度。这个"玩具"版本能让你直观理解 RAG 的每一个环节是如何串联的。

学习目标 理解 RAG 管道的完整数据流，为后续引入 ChromaDB、SSE 流式输出、文档管理等生产级特性打下坚实基础。简单不代表无用——这就是 RAG 的本质骨架。

Step 2：文本分块与向量化

将知识片段向量化——每条片段转换为一个 1536 维的浮点数向量：

Python — 向量化知识库 
from openai import OpenAI
import numpy as np

client = OpenAI()

def embed_texts(texts):
    """向量化一批文本"""
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=texts
    )
    return np.array([item.embedding for item in response.data])

# 将整个知识库向量化
doc_vectors = embed_texts(knowledge_base)
print(f"向量矩阵形状: {doc_vectors.shape}")  # (10, 1536)

至此，我们将 10 条文本转换为了一个 10 × 1536 的 NumPy 矩阵——每一行是一个知识片段的"语义坐标"。

04

Step 3：检索最相关片段

当用户提出问题时，将问题向量化，然后在知识库的向量矩阵中找出最相似的 Top-K 个片段：

Python — 检索函数 
def retrieve(query, knowledge_base, doc_vectors, k=3):
    """
    检索与查询最相关的 Top-K 个知识片段
    
    参数:
        query:          用户问题 (str)
        knowledge_base: 原始知识片段列表
        doc_vectors:    知识片段的向量矩阵 (n, 1536)
        k:              返回结果数量
    """
    # 1. 向量化用户问题
    query_vec = embed_texts([query])[0]

    # 2. 计算余弦相似度
    norms = np.linalg.norm(doc_vectors, axis=1)
    query_norm = np.linalg.norm(query_vec)
    similarities = np.dot(doc_vectors, query_vec) / (norms * query_norm)

    # 3. 获取 Top-K 索引
    top_k_indices = np.argsort(similarities)[-k:][::-1]

    # 4. 返回结果
    results = []
    for idx in top_k_indices:
        results.append({
            'content': knowledge_base[idx],
            'score': float(similarities[idx])
        })
    return results


# 测试检索
query = "谁提出了Transformer架构？"
results = retrieve(query, knowledge_base, doc_vectors, k=3)
for i, r in enumerate(results):
    print(f"[{i+1}] 相似度: {r['score']:.4f}")
    print(f"    内容: {r['content'][:80]}...\n")

当用户问"谁提出了 Transformer 架构？"时，第 3 条知识片段（Transformer 相关内容）的相似度会显著高于其他片段。

Step 4：构建 RAG Prompt 并调用 LLM

将检索到的片段拼入一个结构化的 Prompt，然后调用 LLM 生成最终答案：

Python — RAG 生成 
def rag_generate(query, knowledge_base, doc_vectors, k=3):
    """执行完整的 RAG 流程：检索 → 构建 Prompt → 生成回答"""

    # 1. 检索相关文档片段
    retrieved = retrieve(query, knowledge_base, doc_vectors, k=k)

    # 2. 拼接上下文
    context = "\n---\n".join(
        f"[来源{i+1}] {r['content']}"
        for i, r in enumerate(retrieved)
    )

    # 3. 构建完整的 RAG Prompt
    prompt = f"""你是一个智能助手。请基于以下提供的知识库内容回答用户的问题。
如果知识库中没有相关信息，请如实告知，不要编造答案。

## 知识库内容
{context}

## 用户问题
{query}

## 回答要求
- 基于知识库内容作答，不要使用知识库之外的信息
- 在回答中标注信息来源编号，如 [来源1]
- 如果知识库内容不足以回答问题，请明确说明"""

    # 4. 调用 LLM 生成回答
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.3,
    )
    return response.choices[0].message.content


# 完整流程
answer = rag_generate("谁提出了Transformer架构？", knowledge_base, doc_vectors)
print(answer)
Prompt 设计要点 RAG Prompt 需要明确指示 LLM：基于提供的上下文回答、标注引用来源、不要编造。这三条指令是抑制幻觉的关键。缺少任何一条都可能导致模型"抛开知识库、自由发挥"。

完整可运行代码

以下是整合了所有步骤的完整脚本。复制这段代码，设置好 OPENAI_API_KEY 环境变量即可运行：

📄 完整代码：minimal_rag.py（点击展开/收起） ▼
Python — minimal_rag.py 
"""
最简 RAG 管道示例
不依赖任何向量数据库，纯 Python + NumPy + OpenAI API
"""
import os
import numpy as np
from openai import OpenAI

# ============================================================
# 配置
# ============================================================
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
EMBEDDING_MODEL = "text-embedding-3-small"
LLM_MODEL = "gpt-4o-mini"
TOP_K = 3

# ============================================================
# 知识库（在实际项目中来自 PDF 文件）
# ============================================================
knowledge_base = [
    "深度学习是机器学习的一个分支，使用多层神经网络学习数据的层次化表示。",
    "反向传播算法由Rumelhart、Hinton等人在1986年提出，利用链式法则计算每一层的梯度。",
    "Transformer架构由Vaswani等人在2017年的论文《Attention Is All You Need》中提出，基于自注意力机制。",
    "Python是由Guido van Rossum于1991年创建的通用编程语言，以简洁优雅著称。",
    "RAG全称Retrieval-Augmented Generation，由Facebook AI Research于2020年提出。",
    "卷积神经网络(CNN)特别适合处理图像数据，通过卷积核提取局部特征。",
    "GPT系列模型由OpenAI开发，GPT-4是目前最先进的大语言模型之一。",
    "向量数据库如ChromaDB专门用于存储和检索高维嵌入向量。",
    "自注意力机制允许模型在处理每个词时关注输入序列中的所有其他词。",
    "余弦相似度是衡量两个向量方向一致性的常用方法，广泛应用于信息检索。",
]


def embed_texts(texts):
    """向量化文本"""
    resp = client.embeddings.create(model=EMBEDDING_MODEL, input=texts)
    return np.array([d.embedding for d in resp.data])


def retrieve(query, doc_vectors, k=TOP_K):
    """检索最相关的 K 个知识片段"""
    qv = embed_texts([query])[0]
    norms = np.linalg.norm(doc_vectors, axis=1)
    sims = np.dot(doc_vectors, qv) / (norms * np.linalg.norm(qv))
    top_idx = np.argsort(sims)[-k:][::-1]
    return [(knowledge_base[i], float(sims[i])) for i in top_idx]


def ask_rag(query):
    """执行完整的 RAG 问答流程"""
    retrieved = retrieve(query, doc_vectors)
    context = "\n---\n".join(
        f"[来源{i+1}] {text}" for i, (text, _) in enumerate(retrieved)
    )

    prompt = f"""基于以下知识库内容回答问题，标注引用来源：

{context}

问题：{query}

请基于以上知识库内容回答。如果知识库不足以回答，请明说。"""

    resp = client.chat.completions.create(
        model=LLM_MODEL,
        messages=[{"role": "user", "content": prompt}],
        temperature=0.3,
    )
    return resp.choices[0].message.content


# ============================================================
# 运行演示
# ============================================================
if __name__ == "__main__":
    print("正在向量化知识库...")
    doc_vectors = embed_texts(knowledge_base)
    print(f"✓ 知识库向量化完成，矩阵形状: {doc_vectors.shape}\n")

    questions = [
        "谁提出了Transformer架构？",
        "什么是指反向传播算法？",
        "Python是哪一年创建的？",
    ]

    for q in questions:
        print(f"\n{'='*60}")
        print(f"❓ 问题: {q}")
        print(f"{'='*60}")
        answer = ask_rag(q)
        print(f"🤖 回答:\n{answer}")

对比实验：RAG vs 纯 LLM

对同一个问题，分别使用 RAG 增强模式和纯 LLM 模式进行回答，观察差异：

✅ RAG 增强回答

问：反向传播算法是谁提出的？

"根据您提供的知识库内容，反向传播算法由 Rumelhart、Hinton 等人于 1986 年提出 [来源2]。该算法利用链式法则来计算神经网络中每一层的梯度，是训练深层神经网络的基石。"

⚠ 纯 LLM 回答（无 RAG）

问：反向传播算法是谁提出的？

"反向传播算法的概念可以追溯到 1960 年代。现代形式的反向传播算法主要归功于 David Rumelhart、Geoffrey Hinton 和 Ronald Williams 在 1986 年发表的论文。该算法通过计算损失函数对各层参数的梯度来更新神经网络权重……"

关键差异

RAG 回答明确引用了知识来源（[来源2]），回答简洁聚焦，紧扣知识库内容。纯 LLM 回答虽然也正确，但引用了知识库之外的额外细节（Ronald Williams 的名字），且无法验证信息来源的准确性。

这就是 RAG 的核心优势：可溯源性（Traceability）与可控性（Controllability）——用户可以追溯每条信息的来源，开发者可以通过更新知识库来精确控制 LLM 能"知道"什么。

从玩具到项目：演进路线

这个"玩具版"RAG 与我们的项目级实现（app/services/rag_pipeline.py）之间存在哪些关键差异？

维度 玩具版（本课） 项目版（后续课程）
向量存储 NumPy 数组（内存） ChromaDB（持久化 + 近似搜索）
计算方式 同步（阻塞） FastAPI + asyncio（异步并发）
输出方式 一次性返回全部文本 SSE 流式输出（逐 token 推送）
文档来源 手写知识片段 PDF 上传 → 自动解析分块
引用格式 简单的 [来源N] 标注 结构化的 citations 数组（含文档名、页码、片段）

虽然项目版更强大，但核心逻辑完全一致：向量化 → 检索 → 增强 → 生成。理解了玩具版，你就理解了 RAG 的本质。

本课小结

  • 用纯 Python + NumPy + OpenAI 就能构建完整的 RAG 管道，无需任何向量数据库
  • RAG 管道的四步：准备知识库 → 向量化 → 检索 Top-K → 构建 Prompt 并调用 LLM
  • 检索到的文档片段通过结构化 Prompt 注入上下文，LLM 据此生成可溯源、可验证的回答
  • RAG 回答比纯 LLM 回答更准确、可信、可追溯，因为它有明确的知识来源
  • 项目版的 rag_pipeline.py 在核心逻辑上与玩具版一致，只是加入了 ChromaDB、异步、SSE 等生产级特性
  • 掌握了玩具版，你就掌握了 RAG 的本质——后续课程只是在此基础上不断加砖添瓦
