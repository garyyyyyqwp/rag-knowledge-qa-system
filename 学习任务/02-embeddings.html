Embedding 与向量化：文本到向量的魔法

理解 Embedding 的核心概念，掌握 OpenAI Embedding API 的调用方法

01

什么是 Embedding？

Embedding（嵌入）是将文本转换为固定长度的浮点数向量的过程。这个向量是文本在高维语义空间中的坐标表示。核心规律是：语义相近的文本，其向量的向量距离也更近。

举个例子：假设我们用 1536 维向量表示一句话，"今天天气很好"和"今日气候宜人"的向量会非常接近，而它们与"我喜欢吃苹果"的向量距离则很远。

词嵌入（Word Embedding）

将单个词语映射为一个稠密向量。经典模型如 Word2Vec、GloVe。词嵌入能捕捉词与词之间的关系，经典的例子是：king - man + woman ≈ queen。

句嵌入（Sentence Embedding）

将整个句子或段落映射为一个向量。本项目使用的 OpenAI Embedding API 即属于此类。它整合了整段文本的语义信息，而非对每个词分别编码。

关键数据 text-embedding-3-small 输出 1536 维向量，text-embedding-3-large 输出 3072 维向量。维度越高，表达能力越强，但成本也越高。
02

Embedding API 调用实战

使用 OpenAI 的 Embedding API 非常简单，只需几行代码：

Python — 基础调用 
from openai import OpenAI

client = OpenAI()
response = client.embeddings.create(
    model="text-embedding-3-small",
    input=["今天天气很好", "今日气候宜人", "我喜欢吃苹果"]
)
vectors = [item.embedding for item in response.data]

# 验证: vectors[0] 与 vectors[1] 的余弦相似度应远高于 vectors[0] 与 vectors[2]
print(len(vectors[0]))  # 1536

本项目在 app/services/embedding.py 中封装了更实用的异步方法：

Python — 项目中的实际实现 
async def embed_texts(texts: list[str]) -> list[list[float]]:
    """对一批文本进行向量化，返回对应的向量列表"""
    client = _get_embedding_client()
    response = await client.embeddings.create(
        model=EMBEDDING_MODEL,
        input=texts,
    )
    return [item.embedding for item in response.data]


async def embed_single(text: str) -> list[float]:
    """对单段文本进行向量化"""
    results = await embed_texts([text])
    return results[0]
注意事项 Embedding API 按 token 数量计费。text-embedding-3-small 的价格约为 $0.02 / 1M tokens，性价比极高。生产环境中应避免对同一文本重复计算向量，尽量缓存结果。
03

Embedding 模型对比

截至目前，OpenAI 提供了三款主流 Embedding 模型，各有侧重：

模型 维度 适用场景 价格（每 1M tokens）
text-embedding-3-small 1536 效率优先：大批量文档处理、实时检索、成本敏感场景 $0.02
text-embedding-3-large 3072 精度优先：对检索准确率要求极高的场景 $0.13
text-embedding-ada-002 1536 旧版兼容：遗留系统迁移，不建议新项目使用 $0.10

text-embedding-3-small 是绝大多数 RAG 项目的首选：它在价格和精度之间取得了极佳的平衡，1536 维的向量也非常适合 ChromaDB 使用。

04

可视化：向量空间的直觉

虽然我们无法直接可视化 1536 维空间，但通过降维（如 PCA 或 t-SNE）到 2D，可以看到语义相近的句子在向量空间中确实聚在一起：

语义相近的文本在向量空间中形成集群，不同主题的文本彼此远离。这就是 RAG 检索能够"找到相关内容"的根本原理。

降维可视化工具 实际项目中，可以用 scikit-learn 的 PCA 或 UMAP 将高维向量降维到 2D 或 3D 进行可视化分析。这有助于判断向量质量、发现知识盲区、优化分块策略。
05

项目中的 Embedding 配置

在我们的 FastAPI 项目中，Embedding 相关配置统一管理在 app/utils/config.py 中：

.env 环境变量 
# Embedding 配置
EMBEDDING_PROVIDER=openai
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_API_KEY=sk-xxx    # 默认复用 OPENAI_API_KEY
EMBEDDING_BASE_URL=https://api.openai.com/v1
多平台兼容 由于 OpenAI Embedding API 已成为行业标准，许多平台都提供了兼容接口。你可以通过修改 EMBEDDING_BASE_URL 和 EMBEDDING_API_KEY 切换到 DeepSeek、阿里云 DashScope、腾讯混元 等国内平台的 Embedding 服务，只需确保 API 格式兼容 OpenAI 即可。

这种灵活的设计意味着你的 RAG 系统不会因为某个平台的 API 政策变化而瘫痪——随时可以切换到成本更低或效果更好的 Embedding 服务。

本课小结

  • Embedding 将文本转换为固定长度的浮点数向量，语义相近的文本向量距离更近
  • OpenAI 提供三款 Embedding 模型：text-embedding-3-small（1536维，推荐）、text-embedding-3-large（3072维）、text-embedding-ada-002（旧版）
  • 本项目封装了 embed_texts() 和 embed_single() 两个异步方法，支持批量向量化
  • 向量空间中的距离规律是 RAG 检索的理论基础：相关文本聚在一起，无关文本远离
  • 通过修改环境变量，可以灵活切换到不同平台的 Embedding 服务
