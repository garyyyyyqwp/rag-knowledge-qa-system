文档解析：PDF / Markdown / TXT 处理

学习用 pdfplumber 提取 PDF 文本，理解项目中的文档解析管道

01

为什么需要文档解析？

用户上传的是原始文件（PDF / MD / TXT），RAG 系统需要提取其中的纯文本才能进行后续的切分、Embedding 和入库操作。

PDF

二进制格式，需要专门的解析库（pdfplumber）提取文字层。直接读取只有乱码。

Markdown

纯文本格式，但要保留标题层级（# / ## / ###）等结构信息，以便后续语义切分。

TXT

最简格式，UTF-8 解码即可。但需要注意编码检测和容错处理。

本项目支持 3 种格式：PDF、Markdown（.md）、TXT（.txt）。所有解析结果统一为纯文本字符串，便于下游处理。

02

PDF 解析：pdfplumber

为什么选择 pdfplumber？

库特点本项目的选择
PyPDF2 / pypdf 基础文本提取，速度较快 精度不如 pdfplumber
pdfplumber 更好的文本提取精度，支持表格识别 ✅ 本项目使用
PyMuPDF (fitz) 速度最快、功能最强 但许可证限制，暂不采用
Python 
import pdfplumber
import io

def parse_pdf(content: bytes) -> str:
    text_parts = []
    with pdfplumber.open(io.BytesIO(content)) as pdf:
        for page in pdf.pages:
            page_text = page.extract_text()
            if page_text:
                text_parts.append(page_text)
    return "\n\n".join(text_parts)

注意事项：pdfplumber 无法识别扫描件中的文字（需配合 OCR 如 Tesseract），也无法提取图片中的信息。对于扫描版 PDF，文本提取结果可能为空。

03

Markdown / TXT 解析

Markdown 和 TXT 本质上是纯文本格式，解码即可：

Python 
def parse_markdown(content: bytes) -> str:
    return content.decode("utf-8", errors="replace")


def parse_txt(content: bytes) -> str:
    return content.decode("utf-8", errors="replace")

关于 errors="replace"：

  • 默认行为（errors="strict"）：遇到无法解码的字节直接抛出 UnicodeDecodeError，导致整个解析失败。
  • 容错行为（errors="replace"）：用 U+FFFD 替换字符代替非法字节序列，确保解析不中断。

设计考量：对于学习型项目，"replace" 是合理的妥协——宁可保留少量乱码字符，也不应因为编码问题导致整个文档无法入库。

04

项目完整代码：parser.py

Python 
from pathlib import Path
from typing import Set

SUPPORTED_EXTENSIONS: Set[str] = {".pdf", ".md", ".txt"}
MAX_FILE_SIZE_BYTES = 10 * 1024 * 1024  # 10 MB


class ParseError(Exception):
    """文档解析基类异常"""
    pass


class UnsupportedFileTypeError(ParseError):
    """不支持的文件类型"""
    pass


class FileTooLargeError(ParseError):
    """文件过大"""
    pass


def validate_file(filename: str, content: bytes) -> Path:
    """验证文件扩展名和大小，返回 Path 对象"""
    file_path = Path(filename)
    suffix = file_path.suffix.lower()

    if suffix not in SUPPORTED_EXTENSIONS:
        raise UnsupportedFileTypeError(
            f"不支持的文件类型: {suffix}，"
            f"支持的类型: {', '.join(SUPPORTED_EXTENSIONS)}"
        )

    if len(content) > MAX_FILE_SIZE_BYTES:
        raise FileTooLargeError(
            f"文件大小 {len(content)} 超过限制 {MAX_FILE_SIZE_BYTES}"
        )

    return file_path


async def parse_document(filename: str, content: bytes) -> str:
    """主入口：验证 → 解析 → 返回纯文本"""
    file_path = validate_file(filename, content)

    suffix = file_path.suffix.lower()
    if suffix == ".pdf":
        text = parse_pdf(content)
    elif suffix == ".md":
        text = parse_markdown(content)
    elif suffix == ".txt":
        text = parse_txt(content)
    else:
        raise UnsupportedFileTypeError(f"不支持的文件类型: {suffix}")

    if not text or not text.strip():
        raise ParseError("文档内容为空，无法提取有效文本")

    return text

06

边界处理

空文件检测

if not text or not text.strip(): raise ParseError("文档内容为空")——阻止空文件进入后续管道。

文件大小限制

MAX_FILE_SIZE_BYTES = 10 * 1024 * 1024（10 MB），防止超大文件耗尽内存或导致 API 超时。

不支持的类型

返回 400 Bad Request 并附带清晰错误信息，告知用户当前支持的文件格式列表。

编码容错

errors="replace" 防止因个别乱码字节导致整个文件解析崩溃，确保系统鲁棒性。

📝 动手实验

  1. 准备一个 PDF 文件，用 pdfplumber.open("your_file.pdf") 逐页提取文本，检查提取质量。观察表格、多栏排版的提取效果。
  2. 尝试上传一个不支持的文件类型（如 .docx），观察错误处理流程和返回的 400 状态码。
  3. 向 parser.py 添加对 .html 文件的支持（提示：用 BeautifulSoup 的 get_text()）。

本课要点

  • 文档解析是 RAG 管道的第一步——将原始文件转换为纯文本供后续处理
  • PDF 用 pdfplumber 提取，精度优于 PyPDF2，但无法处理扫描件
  • Markdown / TXT 用 utf-8 解码，errors="replace" 做编码容错
  • 项目分三层异常体系：ParseError → UnsupportedFileTypeError / FileTooLargeError
  • 完整流程：上传 → 验证 → 解析 → 纯文本 → chunker
  • 边界处理：空文件检测、10MB 大小限制、不支持类型的友好错误提示
